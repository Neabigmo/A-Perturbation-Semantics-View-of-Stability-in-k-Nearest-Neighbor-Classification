# A Perturbation-Semantics View of Stability in k-NN

This repository contains the reproducibility code package for the manuscript
*A Perturbation-Semantics View of Stability in k-Nearest Neighbor
Classification*.

The project studies a finite-sample distinction between leave-one-out (LOO)
diagnostics and replace-one robustness for deterministic binary k-nearest
neighbor classifiers. The main implementation fixes:

- ordered finite samples with duplicate occurrences;
- deterministic neighbor ordering by distance and then occurrence index;
- deterministic binary vote ties resolved as `0 < 1`;
- delete-one, insert-one, replace-one, and LOO perturbation indicators;
- witness searches and benchmark experiments for the LOO/replace-one gap.

## Layout

- `src/knn_stability/`: core finite-metric, deterministic k-NN, tie-breaking,
  perturbation, stability, diagnostic, and witness utilities.
- `experiments/`: witness search, synthetic graph-metric benchmarks, tabular
  benchmarks, table generation, and the reproducibility runner.
- `experiments/configs/`: full and smoke-test TOML configurations.
- `tests/`: regression tests for the core definitions and witness behavior.
- `data_tables/`: raw benchmark JSON files and CSV aggregate tables used for
  the manuscript empirical summary.
- `outputs_manifest/`: manifest, config, hashes, and reproducibility notes from
  the archived `ml_submission` run.
- `docs/literature/LIT_TABLE.md`: literature-control table used by the table
  renderer.

The final manuscript figure-rendering script is not included here. The released
code package is focused on the definitions, experiments, tables, and data used
to support the results.

## Environment

Create an environment with conda:

```powershell
conda env create -f environment.yml
conda activate knn-stability-ml
```

Or install from the local package metadata:

```powershell
python -m pip install -e .
```

If network access requires the local proxy used during manuscript preparation:

```powershell
$env:HTTP_PROXY='http://127.0.0.1:7897'
$env:HTTPS_PROXY='http://127.0.0.1:7897'
```

## Run Tests

```powershell
python -m pytest -q
```

## Reproduce Experiments

Smoke run:

```powershell
python experiments\run_all.py --config experiments\configs\ml_submission_smoke.toml
```

Full run:

```powershell
python experiments\run_all.py --config experiments\configs\ml_submission.toml
```

By default, `run_all.py` regenerates witness JSON, synthetic/tabular benchmark
JSON, tables, and a run manifest. Figure rendering is optional and intentionally
excluded from this code package; if a local figure renderer is added at
`experiments/nature_figures.py`, pass `--with-figures`.

## Data Sources

Synthetic benchmark data are generated by `experiments/stability_gap_synthetic.py`
from finite graph metrics. The graph metric is the shortest-path metric on
sampled connected graphs; duplicate ratio, conflict ratio, label noise, sample
size, and `k` are controlled by the TOML config.

Tabular benchmark data are generated by
`experiments/stability_gap_tabular.py` using public datasets bundled with
scikit-learn: Iris, Wine, Breast Cancer, and Digits. The experiment converts
these datasets to binary classification tasks as implemented in the script,
uses seeded subsamples, and reports LOO/replace-one separation frequency and
vulnerable-query rate.

The archived benchmark artifacts in `data_tables/` come from:

```text
outputs/runs/ml_submission/finalrun_manifest/raw/experiments/
```

The corresponding command log, config, file list, and SHA-256 checksums are in
`outputs_manifest/run_manifest.json`.

## Main Commands Used for the Archived Run

The archived run used the `ml_submission` config:

```powershell
python experiments\run_all.py --config experiments\configs\ml_submission.toml --timestamp finalrun_manifest
```

The exact step commands and checksums are recorded in
`outputs_manifest/run_manifest.json`.
